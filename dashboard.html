Slack Connection Instructions

Follow these steps to connect your Rover agent to your Slack account on your phone and start giving instructions.

1. Initial Configuration

  1. Install Slack: Ensure the Slack app is installed on your phone and you are logged in.
  2. Verify Tokens: Open config.env in the project root. Ensure the following are set correctly:
    • SLACK_BOT_TOKEN: Starts with xoxb-
    • SLACK_APP_TOKEN: Starts with xapp-
  3. Restart Rover: Run ./StartRover.sh in your terminal to apply any configuration changes.

2. How to try it

  1. Join the Channel: Go to the Slack channel where you added the @Rover bot (e.g., #general or a private channel). If the bot isn't there, type /invite @Rover.
  2. Send a Ping: Type @Rover ping! in the channel.
  3. Give a Command: You can also send instructions like @Rover check the disk space or @Rover sync the repository.

3. What to expect

  • Immediate Response: For a "ping!", the Rover will reply with pong! 🏓 and the current system time.
  • Task Creation: For any other instruction, the Rover will reply with 🚀 Instruction received... Task created: slack_xxx.md.
  • Action: The agent system will pick up the new task from the tasks/ folder and begin execution in the next cycle.

4. Troubleshooting

If @Rover doesn't respond:

  1. Check if the SlackBridge agent is marked as "Active" in the Dashboard tab.
  2. Verify that Socket Mode is enabled in your Slack App settings (api.slack.com).
  3. Ensure App Mentions are enabled in "Event Subscriptions".
  4. Check the logs: tail -f logs/diary.log | grep SlackBridge
